## 一、项目概述

### 1.1 背景

本看板复现了抖音博主「土豆看财报」提出的**「宝妈指数」**投资分析框架，通过量化手段追踪A股各板块的拥挤度与估值状态，帮助投资者判断当前市场各板块所处周期阶段。

**核心问题**：当某个板块涨到「宝妈」群体（指投资经验较少的散户）都在谈论时，往往意味着阶段性顶部。反之，当所有人都在回避某个板块时，往往是布局机会。

### 1.2 功能目标

| 模块 | 内容 |
|:---|:---|
| 拥挤度地图 | 8个板块 × 6个维度 热力图，直观展示各板块热度 |
| 估值对比表 | PE/PB 历史百分位，对比各板块估值高低 |
| 板块总分排名 | 横向柱状图，量化各板块综合热度 |
| 五维雷达图 | 芯片ETF五维指标雷达图 |
| 净值走势图 | 芯片ETF历史净值曲线，标注关键价位 |
| 持仓结构图 | 芯片ETF前十大持仓及细分方向占比 |

## 三、核心计算逻辑

### 3.1 宝妈指数（核心公式）

宝妈指数总分由**六维度加权综合评分**计算得出：

```
宝妈指数总分 = ORE×30% + 扩散力×15% + 动摇度×15% + D回补力×20% + 有拥挤度×10% + 估值修正×10%
```

各维度含义：

| 维度 | 简称 | 权重 | 计算方式 | 数据来源 |
|:---|:---|:---:|:---|:---|
| ORE拥挤度 | ORE | 30% | 成分股相关性均值，反映同涨同跌程度 | 东方财富/同花顺 |
| 扩散力 | dif | 15% | 上涨成分股占比（%），反映上涨广度 | K线数据统计 |
| 动摇度 | wov | 15% | 近20日波动率，反映价格反复程度 | 日收益率标准差×√252 |
| D回补力 | dbu | 20% | 资金净流入强度（主力净流入/总成交额） | 资金流向数据 |
| 有拥挤度 | cro | 10% | 与大盘相关性，衡量板块与指数同步程度 | 相关系数 |
| 估值百分位 | val | 10% | PE或PB历史分位，衡量估值高低 | westock-data |

### 3.2 拥挤度评分规则（热力图颜色）

| 分数区间 | 颜色 | 信号含义 |
|:---:|:---:|:---|
| > 100 | 🔴 红色 | 情绪极端，风险极高，可能处于泡沫顶 |
| 80 ~ 100 | 🟡 黄色 | 偏热，需警惕，注意保护利润 |
| < 80 | 🟢 绿色 | 正常区间，可以关注 |

### 3.3 估值百分位评分

| PE/PB百分位 | 颜色 | 含义 |
|:---:|:---:|:---|
| ≥ 99% | 🔴 红 | 历史最高估值，极度泡沫化 |
| 95% ~ 99% | 🟡 黄 | 偏高估，建议谨慎 |
| 50% ~ 95% | 🟢 绿 | 正常区间 |
| < 50% | 🔵 蓝 | 偏低估，可能有机会 |

### 3.4 预期底部计算

根据2026年7月16日数据，芯片ETF（159995）的关键价位分析：

| 价位 | 含义 |
|:---|:---|
| 3.3724 | 历史高点（2026-06-30） |

## 四、各模块详解

### 4.1 拥挤度地图（热力图）

**数据结构**：`heatmapData` 数组，每个元素代表一个板块：

```javascript
{
  name: 'AI',        // 板块名称
  ore: 105,          // ORE拥挤度
  dif: 92,          // 扩散力
  wov: 112,         // 动摇度
  dbu: 35,          // D回补力
  cro: 90,          // 有拥挤度
  total: 91,        // 综合总分
  chg: +12          // 环比变化
}
```

**渲染逻辑**：

1. 读取 `heatmapData` 数组
2. 对每个板块的每个维度调用 `heatColor(v)` 判断颜色
3. 总分列单独用 `score-total` 样式渲染，显示分数+环比变化
4. 热力图使用 CSS Grid 布局（7列：板块名+5个维度+总分）

### 4.2 估值对比表

**数据结构**：`valuationData` 数组：

```javascript
{
  name: '半导体设备',  // 板块名称
  pe: 166.96,          // PE-TTM
  pePct: 96.36,        // PE历史百分位（%）
  pbPct: 99.36,        // PB历史百分位（%）
  peChg: -7.5          // PE周环比变化
}
```

**数据来源**：通过 `westock-data` 工具调用，可直接获取各板块/ETF的PE-TTM、PB及历史百分位数据。

**渲染逻辑**：

1. PE/PB百分位 ≥ 95% → 红色；80%~95% → 黄色；< 80% → 绿色
2. PE周环比下降（负数）→ 绿色（估值压力缓解）；上升 → 红色

### 4.3 板块总分横向柱状图

- 使用 Chart.js `bar` 类型，`indexAxis: 'y'` 实现横向
- 颜色逻辑：总分≥90 → 红色；70~90 → 黄色；<70 → 绿色
- Tooltip 显示环比变化

### 4.4 五维雷达图

展示芯片ETF（159995）在五个维度上的得分：

```javascript
radarData = {
  labels: ['动摇度(波动)', '扩散力(涨跌广度)', 'D回补力(资金净流入)',
            '有拥挤度(相关性)', 'PE历史%位(估值)', '宝妈指数总分'],
  values: [95, 85, 40, 92, 94, 91],
  max: 120
}
```

### 4.5 净值走势图（折线图）

- X轴：日期（2026-02 ~ 2026-07）
- Y轴：净值
- 数据点颜色：净值≥3.0 → 红；≥2.0 → 黄；<2.0 → 蓝
- 标注预期底部线（1.10），虚线绿色

## 五、数据源说明

### 5.1 数据源可用性汇总

| 维度 | 理想数据源 | 实际可用方案 |
|:---|:---|:---|
| PE/PB历史百分位 | wind/Bloomberg | ✅ westock-data（直接可用） |
| 成分股拥挤度 | 高级另类数据 | ✅ 东方财富/同花顺成分股权重 |
| K线数据 | wind | ✅ 东方财富API |
| 资金流向 | wind | ✅ 东方财富资金流API |
| 主力净流入 | Level2 | ✅ eastmoney API（免费） |
| 机构持仓结构 | 基金季报 | ✅ eastmoney 基金档案页（浏览器抓取） |
| 社媒讨论热度 | 雪球/抖音/百度指数 | ❌ 无免费API，需登录态 |
| ETF持有人结构 | 基金档案 | ⚠️ 需浏览器抓取（非API） |

### 6.2 GitHub Actions 工作流

**触发方式**（三种）：

| 触发方式 | cron表达式（UTC） | 对应北京时间 |
|:---|:---|:---|
| 定时触发① | `50 3 * * *` | 北京时间 11:40 |
| 定时触发② | `50 5 * * *` | 北京时间 13:10 |
| repository_dispatch | 自定义 `dispatch_update` 事件 | 手动触发 |
| workflow_dispatch | — | GitHub 网页手动运行 |

**工作流核心步骤**：

1. **Checkout**：克隆仓库
2. **获取最新数据**：通过东方财富API抓取当日ETF净值、资金流向等
3. **更新HTML**：将最新数据注入 `babymom-dashboard.html` 的 JavaScript 数据变量
4. **推送更新**：将更新后的文件推送回仓库
5. **触发 Pages 重建**：GitHub Pages 自动检测到新提交并重新部署

### 6.3 关键技术教训

> ⚠️ **教训1：workflow文件必须放在 `.github/workflows/` 目录**
>
> GitHub Actions 只识别 `.github/workflows/` 下的 YAML 文件，放在根目录无法触发。

> ⚠️ **教训2：workflow文件名不能重复**
>
> GitHub API 对同名 workflow 文件有缓存，直接 PUT 覆盖会报 422 错误。解决：创建新文件名（如 `daily-update-v2.yml`）。

> ⚠️ **教训3：Git Trees API 的 base_tree 问题**
>
> 通过 Git Data API 更新 HEAD 时，必须手动构建完整的 blob + tree 结构，不能只传 base_tree。

### 6.4 GitHub Personal Access Token 管理

| Token | 状态 | 说明 |
|:---|:---:|:---|
| `ghp_5O…MBFb` | ❌ 已失效 | 权限不足（缺少workflow scope） |
| `ghp_8F…FVZ0` | ❌ 已失效 | — |
| `ghp_btX…` | ✅ 当前有效 | 完整 repo + workflow 权限 |

> 📌 Token 有效期需持续监控，过期前在 GitHub Settings → Developer settings → Personal access tokens 续期。
